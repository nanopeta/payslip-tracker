# 給与明細・源泉徴収票ダッシュボード

OBC 給与システム（hromssp.obc.jp）から保存した MHT ファイルをアップロードして、給与明細・源泉徴収票を管理・可視化するWebアプリ。

**URL**: https://nanopeta.github.io/payslip-tracker/

## 特徴

- **完全ローカル処理**: ファイル解析・データ保存はすべてブラウザ内のみ。外部サーバーへのデータ送信なし
- **MHT自動解析**: 給与明細・賞与・インセンティブ・源泉徴収票を自動判別してデータ抽出
- **複数ファイル一括アップロード**: 複数の MHT ファイルをまとめてアップロード・確認・保存
- **月次グラフ**: 差引支給額の推移（通常給与のみ・賞与のみを別グラフで表示）、支給vs控除の内訳
- **みなし残業効率**: みなし残業額と実残業額の差額を自動計算（複数の実残業項目に対応）
- **スマホ対応**: モバイルファーストなUI（ボトムナビゲーション）

## 使い方

1. OBC 給与システムで給与明細・源泉徴収票ページを「名前を付けて保存（MHT/MHTML形式）」で保存
2. アップロードタブから MHT ファイルを選択（複数選択可）
3. 自動解析された内容を確認・修正
4. 保存するとダッシュボードに反映される

対応ファイル:
- OBC 給与システムの給与明細（給与・賞与・インセンティブ）
- 給与所得の源泉徴収票

## 画面構成

| ページ | 内容 |
|---|---|
| ダッシュボード | 最新月のサマリ・月次推移グラフ（通常給与／賞与）・みなし残業効率 |
| 給与明細 | 月次一覧・詳細（支給/控除/勤怠）・一括削除 |
| 年次サマリ | 源泉徴収票・給与明細からの年次集計 |
| アップロード | MHTアップロード・自動解析・確認・登録 |
| 設定 | みなし残業ラベル・実残業ラベル（複数設定可）の設定 |

## データの保存について

すべてのデータはブラウザの `localStorage` に保存されます。

- ブラウザのデータを削除するとデータも消えます
- 異なるブラウザ・デバイス間でデータは共有されません
- データをリセットしたい場合は DevTools > Application > Local Storage で `payslip_tracker_v1` を削除

## 開発環境のセットアップ

```bash
npm install
npm run dev     # http://localhost:5173/payslip-tracker/
```

```bash
npm run build   # 本番ビルド（TypeScript チェック + Vite ビルド）
npm run preview # ビルド結果をローカルで確認
```

## 技術スタック

- **フレームワーク**: React 18 + Vite + TypeScript
- **スタイル**: Tailwind CSS v3（3.4.19 固定）
- **グラフ**: Recharts v2
- **状態管理**: Zustand v5
- **ストレージ**: localStorage（バックエンド不要）
- **ルーティング**: React Router v6
- **デプロイ**: GitHub Pages（GitHub Actions）

## デプロイ

`main` ブランチへのプッシュで GitHub Actions が自動ビルド・デプロイします。
