# 給与明細・源泉徴収票ダッシュボード

OBC 給与システム（hromssp.obc.jp）から保存した MHT ファイルをアップロードして、給与明細・源泉徴収票を管理・可視化するWebアプリ。

**URL**: https://nanopeta.github.io/payslip-tracker/

## 特徴

- **完全ローカル処理**: ファイル解析・データ保存はすべてブラウザ内のみ。外部サーバーへのデータ送信なし
- **MHT 自動解析**: 給与明細・賞与・インセンティブ・源泉徴収票を自動判別してデータ抽出
- **複数ファイル一括アップロード**: 複数の MHT ファイルをまとめてアップロード・確認・保存（重複検出付き）
- **月次・年次グラフ**: 支給・手取りの推移（全期間 / 今年 / 6ヶ月 / 12ヶ月 で切替可能）
- **今年の累計（YTD）**: 年間総支給・年間手取り・年間控除合計をダッシュボードで一目確認
- **収支内訳ドーナツチャート**: 支給・控除内訳の割合を月別で可視化（概要 / 支給 / 控除 の3タブ）
- **みなし残業効率**: みなし残業額と実残業額の差額・残業時給・基本時給を自動計算。残業時間推移（45h / 80h 参照線付き）と年間合算も表示
- **給与明細ソート・フィルター・検索**: 年別グループ表示 / 手取り・総支給・日付順ソート / 年月・種別フィルター / フリーテキスト検索
- **同種別前後ナビゲーション**: 給与↔給与、賞与↔賞与で前後比較。前月比（手取り・総支給・控除・勤怠・残業時間）付き
- **ふるさと納税シミュレーター**: 年収試算から上限額を自動計算。iDeCo・生命保険料・扶養人数を考慮（令和7年度税制改正対応）
- **年末調整 推定還付金**: 毎月の源泉徴収合計と正確な年税額の差額を試算
- **データバックアップ・復元**: JSON エクスポート / JSON インポートでデータを完全バックアップ・復元
- **スマホ対応**: モバイルファーストな UI（ボトムナビゲーション）
- **AI エージェントチーム**: PM・Dev・Reviewer の3エージェントがスプリント方式でアプリを自律改善（`npm run agent-team`）

## 使い方

1. OBC 給与システムで給与明細・源泉徴収票ページを「名前を付けて保存（MHT/MHTML 形式）」で保存
2. アップロードタブから MHT ファイルを選択（複数選択可）
3. 自動解析された内容を確認・修正
4. 保存するとダッシュボードに反映される

対応ファイル:
- OBC 給与システムの給与明細（給与・賞与・インセンティブ）
- 給与所得の源泉徴収票

## 画面構成

| ページ | 内容 |
|---|---|
| ダッシュボード | 最新月サマリ・YTD累計・支給手取り推移・みなし残業効率（年間合算付き）・収支内訳3タブ |
| 給与明細 | 月次一覧（ソート / フィルター / フリーテキスト検索・年別グループ表示）・詳細（前後ナビ・前月比・収支内訳）・一括削除 |
| 年次サマリ | 年別の支給・控除・勤怠の年間集計・月別棒グラフ（給与/賞与色分け）・ふるさと納税シミュレーター・年末調整還付金試算・CSV エクスポート |
| アップロード | MHT アップロード・自動解析・確認・登録（重複検出付き） |
| 設定 | みなし残業ラベル設定（試算プレビュー付き）・JSON/CSV エクスポート・JSON インポートでのデータ復元 |

### ダッシュボード詳細

- **差引支給額カード**: 最新月の手取り・総支給・控除合計・手取り率を表示。前月比付き
- **今年の累計（YTD）**: 給与/賞与別の総支給・手取り・控除合計テーブル + 月次平均・最高・最低月
- **収支内訳カード**: 月選択ドロップダウン付きで「概要 / 支給 / 控除」3タブ切替。ドーナツチャートと前月比差額
- **みなし残業効率カード**: 差額・みなし残業額・実残業代・残業時間・使用率（プログレスバー）・残業時給・基本時給を1枚に集約。残業時間推移（棒グラフ・45h/80h 参照線）・月次差額推移・年間合算を内包
- **支給・手取りの推移**: 全種別合計と給与のみの折れ線グラフ。賞与データがあれば自動で給与のみラインを追加表示
- 推移グラフは **全表示 / 今年 / 6ヶ月 / 12ヶ月** でフィルター切替可能

## データの保存について

すべてのデータはブラウザの `localStorage` に保存されます。

- ブラウザのデータを削除するとデータも消えます
- 異なるブラウザ・デバイス間でデータは共有されません
- データをリセットしたい場合は DevTools > Application > Local Storage で `payslip_tracker_v1` を削除
- 設定（みなし残業ラベル）は `payslip_tracker_settings` に別保存されるため、データ削除後も維持されます

## エージェントチーム（AI 自律改善）

PM・Dev・Reviewer の3 Claude エージェントがスプリント方式で協調して、このアプリを自律的に改善します。

```bash
npm run agent-team                          # API キー不要（Claude Code CLI を使用）
npm run agent-team -- --sprint-size=3       # 1スプリント 3件で実行
npm run agent-team -- --max-sprints=5       # スプリント数を指定（デフォルト: 1）
npm run agent-team -- --fast                # Reviewer・ドキュメント更新をスキップ
npm run agent-team -- --no-push             # スプリント完了後のプッシュを省略
npm run agent-team -- --model=claude-haiku-4-5-20251001  # 使用モデルを指定
```

**動作フロー:**
1. **PM** — コードと `git log` を読んで未実装の改善を N 件バックログ化
2. **Dev** — 各タスクを実装して `npm run build` で検証
3. **Reviewer** — `git diff` でレビュー → LGTM or 修正指示（失敗しても続行）
4. タスクごとに自動コミット → スプリント後に `CLAUDE.md` を自動更新
5. 全スプリント完了または `Ctrl+C` でブランチを自動プッシュ

**注意**: 各 Claude CLI 呼び出しのタイムアウトは 10 分。失敗時は 1 回自動リトライします。

## 技術スタック

- **フレームワーク**: React 18 + Vite + TypeScript
- **スタイル**: Tailwind CSS v3（3.4.19 固定）
- **グラフ**: Chart.js v4 + Recharts v2
- **状態管理**: Zustand v5
- **ストレージ**: localStorage（バックエンド不要）
- **ルーティング**: React Router v6
- **AI エージェント**: Claude Code CLI（API キー不要）
- **デプロイ**: GitHub Pages（GitHub Actions）

## デプロイ

`main` ブランチへのプッシュで GitHub Actions が自動ビルド・デプロイします。
